# Jira CSV Generation Guidelines

## Purpose

These guidelines define how to generate Jira Tasks and subtasks that operationalize active marketing campaign Epics for a given month. The output is a Jira-importable CSV that creates Tasks, descriptions, subtasks, and dates under existing Epics.

Jira becomes the operational record for what shipped and when — making "what went out last week" answerable directly from the board.

## Inputs Required

Each run requires:

1. Target timeframe
2. Timeframe-specific context (product launches, events, announcements)

## Jira Structure

### Epics

Epics represent a campaign theme and are long-lived. They are assumed to exist before task generation and are not included in the CSV.

Naming format: `YEAR Theme`
Example: `2026 Foundations Free`

### Tasks

Tasks are the atomic unit of work. Each Task represents one publishable deliverable — completion equals publication.

Naming format: `Lever - Channel - Deliverable Name`
Example: `Social Media - LinkedIn - Foundations Announcement`

Tasks include an auto-generated description with sample text based on the campaign theme and context.

### Subtasks

Subtasks represent production steps and dependencies within a Task.

Naming format: `Channel - Step`
Example: `LinkedIn - Image Creation`

Subtasks include an auto-generated action description.

## Capacity Limits

### Weekly

| Output | Limit |
|---|---|
| Cerkl.com blog post | 1 |
| internalcommspro.com blog post | 1 |
| Social media posts | 4 |
| Marketing email | 1 |
| SEM review task | 1 |

Social media posts break down as:
- 1 LinkedIn static post connected to theme
- 1 LinkedIn static post connected to blog
- 1 LinkedIn poll post
- 1 LinkedIn short video post


## Date Rules

**Start date** is the date work begins on the first production subtask.

**Due date** is the publish date. All subtasks must complete on or before the Task due date.

Social media posts publish Tuesday through Friday by default. All other channels publish on the earliest eligible weekday unless otherwise constrained. Calendar constraints override all default publish rules. Each Task has exactly one publish date.

## Channel Subtask Templates

Subtasks are attached automatically based on channel. No additional subtasks are generated by default.

### Blog Post

Description must include: platform (cerkl.com vs. internalcommspro.com), topic, prompt to generate a draft, theme context.

- Blog – Copy
- Blog – Image Creation
- Blog – Approval
- Blog – Implementation / Publishing

### Social Media

Description must include: topic, post type (static image, poll, video), prompt to generate a draft, theme context.

- [Platform] – Copy
- [Platform] – Asset Creation
- [Platform] – Approval
- [Platform] – Implementation / Publishing

### Marketing Email

- Email – Copy
- Email – Audience Segmentation
- Email – Review
- Email – Schedule Send

### Press Release

- PR – Copy
- PR – Review
- PR – Submit for Release


## Jira Field Mapping

### Task Fields

| Field | Value |
|---|---|
| Issue Type | Task |
| Epic Link | Assigned Epic |
| Summary | `Lever - Channel - Deliverable Name` |
| Description | Generated sample based on theme and context |
| Owner | Channel owner (see Ownership table) |
| Channel | Custom Channel field |
| Start Date | Date work begins |
| Due Date | Publish date |

### Subtask Fields

| Field | Value |
|---|---|
| Issue Type | Subtask |
| Parent | Parent Task summary |
| Summary | `Channel - Step` |
| Description | Generated description based on theme and task topic |

## CSV Output Requirements

- One row per Task and subtask
- Parent-child relationships explicitly defined in each row
- Dates in Jira-compatible format (YYYY-MM-DD)
- Importable without manual cleanup, except for:
  - Adding the ID of the associated Epic
  - Adding IDs for subtask owners

## Reusability

These guidelines apply to every Epic and campaign without modification. Any deviation is an explicit, ad hoc override — not an update to the system.
