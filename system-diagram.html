Cerkl Workspace · Agentic System Reference

How the system routes a request — top to output

No code, no agent classes. Folders are the app; English is the routing language. Every layer loads only what its scope needs. Claude-authored scripts live at the leaf so the LLM doesn't pay tokens for deterministic work.

Cross-cutting flows

⇶
Sub-agent fan-out
One process dispatches N parallel Agent calls in a single message. Each sub-agent gets a self-contained brief. Fallback: parallel tool calls if Agent unavailable.
⇡
Push-update protocol
When marketing affects a PA project, it appends an ## Update — YYYY-MM-DD block to the project file. PA reconciles. Folders never reach across silently.
⇪
Index aggregation
Mission Control + Content Dashboard scan filenames across the workspace and rebuild data.json. No DB, no vector store — naming convention is the index.
{ }
Claude-authored scripts
Deterministic work moves into .py / .sh at the leaf — token-cheap, repeatable, reviewable. The LLM orchestrates; shell executes.

Why the structure holds

  • Token discipline is structural — load only what this layer needs.
  • Router ≠ content doc — if a CLAUDE.md exceeds a screen, it's leaking.
  • Child replaces parent — leaf routers are self-sufficient.
  • Naming convention = database — predictable filenames, no indexes.
  • Skill vs. process — atomic capability vs. multi-step deliverable.
  • The job is the unit of design — every folder ships an outcome.
